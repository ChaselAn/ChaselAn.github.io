Masonry概述

  • Masonry,一个轻量级的布局框架,采用更”优雅”的语法封装自动布局,不需要使用XIB和Storyboard, 并具有高可读性 而且同时支持 iOS 和 Max OS X
  • Masonry尤其适合习惯纯代码开发的开发者 ,在iPhone6发布后引发的适配潮中 Masonry一定可以助你一臂之力
  • 框架下载地址: https://github.com/Masonry/Masonry

卸载约束

  • 需要卸载约束的条件
    • ViewA的某个约束相对于ViewB更改到相对于ViewC
    • ViewA的某个约束相对于View改成具体数值（反之一样）
  • 首先定义两个约束属性
// MASConstraint：约束对象
@property (nonatomic, strong) MASConstraint *cons1;
@property (nonatomic, strong) MASConstraint *cons2;
  • 在viewDidLoad中设置约束
// 添加view
    [self.view addSubview:view3];
    // 添加约束
    [view3 mas_makeConstraints:^(MASConstraintMaker *make) {
        make.top.mas_equalTo(view1.mas_bottom).mas_offset(10);
        self.cons1 = make.left.mas_equalTo(view1);
        self.cons3 = make.width.mas_equalTo(100);
        make.height.mas_equalTo(100);
//        make.size.mas_equalTo(CGSizeMake(100, 100));
    }];
    self.v3 = view3;
  • touchsBegan中进行约束卸载的test
- (void)touchesBegan:(NSSet<UITouch *> *)touches withEvent:(UIEvent *)event{    
    [self.cons3 uninstall];
    [self.cons1 uninstall];

    [self.v3 mas_updateConstraints:^(MASConstraintMaker *make) {
        make.left.mas_equalTo(self.v2);
        make.width.mas_equalTo(self.v1.mas_width).multipliedBy(0.5);
    }];    
}

Masonry的偷懒用法

  • 通过引入MAS_SHORTHAND和MAS_SHORTHAND_GLOBALS两个宏，来更方便的使用Masonry
//define this constant if you want to use Masonry without the 'mas_' prefix
// 如果你想在使用Masonry框架时省略mas_这个前缀,请定义下面这个宏
#define MAS_SHORTHAND
//define this constant if you want to enable auto-boxing for default syntax
//equalTo 如果想让equalTo也带有装箱功能请定义下面这个宏
#define MAS_SHORTHAND_GLOBALS
#warning mark 上面两个宏一定要放在框架得主头文件得上面,它会影响框架得编译
#import "Masonry.h"
  • 引入这两个宏之后，上面的代码中的mas_就都可以省略了
// 省略mas_的版本
[blueView makeConstraints:^(MASConstraintMaker *make) {
    //blueView的左边参照父控件的左边加50
    make.left.equalTo(self.view.left).offset(50);
    //blueView的上边边参照父控件的上边减50
    make.top.equalTo(self.view.top).offset(-50);
    //给blueView上下左右分别设置内边距,上下左右都往里缩50
    make.edges.equalTo(UIEdgeInsetsMake(50, 50, 50, 50));
}
